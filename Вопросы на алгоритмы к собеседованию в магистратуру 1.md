## Шифр первокурсника

#лёгкая

На факультете шифрования Центрального университета первокурсникам выдали необычное задание: научиться мгновенно переворачивать каждое слово в университетских объявлениях, чтобы, по легенде, защитить расписание от злых духов-спойлеров. Порядок слов остаётся неизменным, а вот буквы внутри каждого слова встают задом наперёд. На вход подаётся строка, и вам нужно вернуть её «отзеркаленную» версию, сохранив все пробелы и последовательность слов.

Ваша задача: реализовать алгоритм, который переворачивает каждое слово в строке `s` (слова разделены ровно одним пробелом), не меняя порядка слов. Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти, не считая выходной строки.

### Примеры

| Входные данные                                     | Выходные данные                                |
| -------------------------------------------------- | ---------------------------------------------- |
| `s = "Вот бы поступить в Центральный Университет"` | `"тоВ ыб ьтипутсоп в йыньлортнеЦ тетисревинУ"` |
| `s = "a b c d"`                                    | `"a b c d"`                                    |
| `s = "hello"`                                      | `"olleh"`                                      |

### Решение

- **Идея:** разделяем строку по пробелам на слова, каждое слово переворачиваем с помощью среза `[::-1]` (или двух указателей), затем склеиваем обратно через пробел.
- **Сложность:** O(n) по времени (каждый символ обрабатывается дважды) и O(n) памяти для результирующей строки. Если не учитывать выходную строку, дополнительной памяти используется O(1).

```python
def reverseWords(s: str) -> str:
    return ' '.join(word[::-1] for word in s.split())
```

## Слияние караванов

#легкая

Шейх Халид получил два каравана с золотыми слитками, пронумерованными по весу. Каждый караван уже отсортирован по весу от лёгких слитков к тяжёлым. Первый караван (`nums1`) идёт с запасом — в конце у него достаточно свободных верблюдов (заполненных нулями), чтобы разместить всё золото второго каравана. Шейх приказал слить оба каравана прямо на ходу, не используя лишнего снаряжения, чтобы итоговый караван остался строго отсортированным по весу.

Вам даны два целочисленных массива `nums1` и `nums2`, отсортированные по неубыванию, и два числа `m` и `n` — количество значимых элементов в `nums1` и `nums2` соответственно. Массив `nums1` имеет размер `m + n`, где первые `m` элементов — значимые, а остальные `n` заполнены нулями. Объедините элементы `nums2` в `nums1` **на месте**, чтобы итоговый массив остался отсортированным. Алгоритм должен работать за **O(m + n)** времени и **O(1)** дополнительной памяти.

### Примеры

| Входные данные (nums1, m, nums2, n)               | Результат (модифицированный nums1) |
| ------------------------------------------------- | --------------------------------- |
| `nums1 = [1,2,3,0,0,0], m = 3, nums2 = [2,5,6], n = 3` | `[1,2,2,3,5,6]`                  |
| `nums1 = [1], m = 1, nums2 = [], n = 0`           | `[1]`                            |
| `nums1 = [0], m = 0, nums2 = [1], n = 1`          | `[1]`                            |
| `nums1 = [4,5,6,0,0,0], m = 3, nums2 = [1,2,3], n = 3` | `[1,2,3,4,5,6]`                  |

### Решение

- **Простое решение (O((m+n) log(m+n))):** скопировать `nums2` в конец `nums1` и отсортировать массив целиком. Не требует дополнительной памяти под новый массив, но модифицирует тот же.
- **Оптимальное решение (O(m+n), O(1)):** идём с конца двумя указателями — один на последний значимый элемент `nums1` (индекс `m-1`), второй на последний элемент `nums2` (индекс `n-1`). Сравниваем их и ставим больший в позицию `m+n-1`, сдвигая соответствующий указатель. Так заполняем массив от конца к началу, не затирая необработанные данные.

```python
from typing import List

def merge(nums1: List[int], m: int, nums2: List[int], n: int) -> None:
    i, j, k = m - 1, n - 1, m + n - 1
    while i >= 0 and j >= 0:
        if nums1[i] > nums2[j]:
            nums1[k] = nums1[i]
            i -= 1
        else:
            nums1[k] = nums2[j]
            j -= 1
        k -= 1
    # Если остались элементы в nums2, переносим их
    while j >= 0:
        nums1[k] = nums2[j]
        j -= 1
        k -= 1
```
## Зеркальное Древо

#легкая

В Зачарованном Лесу растёт особое Древо-Близнец. Легенда гласит, что если дерево зеркально симметрично, то оно служит порталом в параллельный мир. Эльфийский старейшина хочет проверить, является ли данное бинарное дерево волшебным порталом — для этого нужно убедиться, что оно симметрично относительно своего центра (левое поддерево является зеркальным отражением правого).

Ваша задача: напишите функцию, которая по корню бинарного дерева `root` определяет, является ли дерево **зеркально симметричным**.

### Примеры

**Пример 1**

```
          1
         / \
        2   2
       / \ / \
      3  4 4  3
```
Выход: `True` — левая половина отражает правую.

**Пример 2**

```
      1
     / \
    2   2
     \   \
      3   3
```
Выход: `False` — правый потомок левого поддерева не равен левому потомку правого поддерева.

**Пример 3**

```
          1
         / \
        2   2
       /     \
      3       3
```
Выход: `True`

### Решение

- **Идея:** рекурсивно сравниваем два поддерева — левое и правое. Два узла зеркальны, если их значения равны, левый потомок левого узла зеркален правому потомку правого, а правый потомок левого — левому потомку правого. Базовые случаи: оба `None` → `True`, один `None` → `False`.
- **Сложность:** O(n) по времени (посещаем каждый узел один раз), O(h) по памяти из-за стека рекурсии, где h — высота дерева.

```python
from typing import Optional

# Определение узла дерева
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def solution(root: Optional[TreeNode]) -> bool:
    def isMirror(left, right):
        if not left and not right:
            return True
        if not left or not right:
            return False
        return (
            left.val == right.val 
            and isMirror(left.left, right.right) 
            and isMirror(left.right, right.left)
        )
    
    if not root:
        return False  # по условию: пустое дерево не считается симметричным
    return isMirror(root.left, root.right)
```



## Страницы пиратского дневника

#легкая

Капитан Блэк нашёл в затопленной каюте потрёпанный дневник старого штурмана. Все уцелевшие страницы пронумерованы, аккуратно отсортированы, и номера уникальны — пират это проверил лично. Теперь ему нужно сжать записи в судовом журнале: выписать все непрерывные диапазоны страниц, чтобы сэкономить чернила и место. Отдельную страницу записывают как сам номер, а диапазон — как «начало → конец». Например, страницы 0, 1, 2, 4, 5 и 7 превратятся в запись `"0->2"`, `"4->5"`, `"7"`.

Ваша задача: получив отсортированный массив уникальных целых чисел `nums`, вернуть список строк, представляющих **минимальный набор непрерывных диапазонов**, покрывающих все числа. Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти (не считая выходного списка).

### Примеры

|Входные данные|Выходные данные|
|---|---|
|`nums = [0,1,2,4,5,7]`|`["0->2","4->5","7"]`|
|`nums = [1,3]`|`["1","3"]`|
|`nums = [1,2,3,5,6,8]`|`["1->3","5->6","8"]`|
|`nums = [10]`|`["10"]`|

### Решение

- **Идея:** линейным проходом находим начало `start` диапазона и, пока следующий элемент отличается от текущего ровно на 1, расширяем конец `j`. После этого записываем диапазон в соответствующем формате и перепрыгиваем на следующий за концом элемент.
- **Сложность:** O(n) по времени, O(1) дополнительной памяти (не считая результирующего списка).

```python
def solution(nums: List[int]) -> List[str]:
    i, result = 0, []
    while i < len(nums):
        start, j = nums[i], i
        while j + 1 < len(nums) and nums[j+1] == nums[j] + 1:
            j += 1
        result.append(f"{start}" if nums[j] == start else f"{start}->{nums[j]}")
        i = j + 1
    return result
```

## Ключ к звёздному шифру

#средняя

Радиотелескоп «Гелиос» перехватил закодированное сообщение от неизвестной цивилизации. Учёные выяснили: сигнал — это строка `s`, и чтобы выделить ключ, нужно найти **самую длинную непрерывную последовательность символов, в которой ни один символ не повторяется**. Длина этой последовательности и есть уровень доступа к следующему слою шифра. Если таких отрезков несколько — годится любой, важна только длина.

Ваша задача: по заданной строке `s` вернуть **длину самой длинной подстроки без повторяющихся символов**. Алгоритм должен работать за **O(n)** времени и использовать не более **O(k)** дополнительной памяти, где `k` — размер алфавита.

### Примеры

| Входные данные | Выходные данные | Пояснение                          |
| -------------- | --------------- | ---------------------------------- |
| `s = "abcabcbb"` | `3`             | Самая длинная подстрока — `"abc"`. |
| `s = "pwwkew"`   | `3`             | Самая длинная подстрока — `"wke"` .|
| `s = ""`         | `0`             | Пустая строка — ничего не принимаем. |
| `s = "aa"`       | `1`             | Только `"a"` без повторов.         |

### Решение

- **Идея:** Используем метод скользящего окна. Поддерживаем левую границу окна `l` и множество `seen` символов текущего окна. Идём правой границей `r` по строке; если символ `s[r]` уже есть в окне, сдвигаем левую границу, удаляя символы, пока дубликат не исчезнет. Размер окна `r - l + 1` всегда содержит только уникальные символы. Обновляем результат максимальной длиной.
- **Сложность:** O(n) по времени — каждый символ добавляется и удаляется не более одного раза; O(k) по памяти, где k — размер алфавита (в худшем случае 26 строчных букв, 52 буквы, цифры и т.д., что константно относительно стандартных ограничений).

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        l, seen, result = 0, set(), 0
        for r in range(len(s)):
            while s[r] in seen and l < r:
                seen.remove(s[l])
                l += 1
            seen.add(s[r])
            result = max(result, r - l + 1)
        return result
```




## Максимальная серия успешных пакетов

#средняя

Руслан копается в логах своего роутера. Лог — это бинарный массив `nums`, где `1` означает успешно доставленный пакет, а `0` — потерю. Руслан решил подчистить историю: он может **удалить ровно одну запись** (любую) из лога, чтобы представить работу сети в лучшем свете. После удаления он хочет узнать, какой максимальной длины будет непрерывная цепочка успешных пакетов.

Ваша задача: получив бинарный массив `nums`, вернуть **длину самой длинной непрерывной подпоследовательности, состоящей только из `1`, после удаления ровно одного элемента** из массива. Если массив состоит только из `1`, всё равно необходимо удалить один элемент.

Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти.

### Примеры

| Входные данные                   | Выход |
| -------------------------------- | ----- |
| `nums = [1,1,0,1,1,1]`          | `5`   |
| `nums = [0,1,1,1,0,1,1,0,1]`   | `5`   |
| `nums = [1,1,1]`                | `2`   |
| `nums = [1]`                    | `0`   |

### Решение

- **Идея:** используем скользящее окно, в котором разрешено не более одного нуля. Размер такого окна `(r - l)` — это и есть длина итоговой цепочки единиц после удаления одного элемента (удаляем либо этот ноль, либо, если нулей нет, одну единицу). Правый указатель `r` движется по массиву; если встречаем ноль, увеличиваем счётчик. Когда нулей в окне становится больше одного, сдвигаем левый указатель `l` до тех пор, пока снова не останется ≤ 1 нуля. На каждом шаге обновляем максимальную длину `max_window = max(max_window, r - l)`.
- **Сложность:** O(n) по времени, O(1) дополнительной памяти.

```python
from typing import List

def oneDeleting(nums: List[int]) -> int:
    l, count, max_window = 0, 0, 0
    for r in range(len(nums)):
        if nums[r] == 0:
            count += 1
        while count > 1 and l < r:
            if nums[l] == 0:
                count -= 1
            l += 1
        max_window = max(max_window, r - l)
    return max_window
```
## Проклятый свиток

#средняя

Археолог Лара обнаружила в заброшенном храме магический свиток с последовательностью чисел. Согласно древней легенде, эта последовательность когда-то была строго возрастающей, но жрецы наложили на неё проклятие сдвига: массив был циклически повёрнут. Чтобы открыть сокровищницу, Лара должна найти **наименьшее число** в этом повёрнутом массиве — именно оно укажет верный код от замка. Все числа в свитке **уникальны**, так что минимум — гарантированно единственный.

Ваша задача: дан отсортированный по возрастанию массив уникальных целых чисел `nums`, который затем был повёрнут (сдвинут) вправо на неизвестное число позиций. Найдите минимальный элемент. Алгоритм должен работать за **O(log n)** времени и использовать **O(1)** дополнительной памяти.

### Примеры

| Входные данные               | Выходные данные |
| ---------------------------- | --------------- |
| `nums = [3,4,5,1,2]`         | `1`             |
| `nums = [4,5,6,7,0,1,2]`     | `0`             |
| `nums = [11,13,15,17]`       | `11`            |
| `nums = [2,1]`               | `1`             |

### Решение

- **Идея:** используем бинарный поиск. Если `nums[mid] > nums[r]`, значит точка разворота (и минимум) лежит строго правее `mid`. Иначе минимум находится в левой половине, включая `mid`.
- **Сложность:** O(log n) по времени, O(1) по памяти.

```python
from typing import List

class Solution:
    def findMin(self, nums: List[int]) -> int:
        l, r = 0, len(nums) - 1
        while l < r:
            mid = l + (r - l) // 2
            if nums[mid] > nums[r]:
                l = mid + 1
            else:
                r = mid
        return nums[l]
```
## Сундуки с золотом

#средняя 

Капитан Блэк раздобыл карту острова, вдоль берега которого в линию зарыты сундуки с золотыми монетами. В каждом сундуке может быть разное количество монет (целое число). Карта указывает, сколько монет в каждом сундуке, но капитана интересуют только непрерывные участки берега, где сумма монет в точности равна заветному числу `k` — именно столько нужно, чтобы снарядить новый корабль. Помогите капитану быстро пересчитать **все возможные непрерывные подмассивы**, сумма элементов которых равна `k`.

Формально: дан целочисленный массив `nums` и целое число `k`. Верните количество непрерывных подмассивов (последовательностей идущих подряд элементов), сумма которых равна `k`.

Алгоритм должен быть эффективным: ожидаемое время **O(n)**, память **O(n)** (из-за хранения хэш-таблицы префиксных сумм).

### Примеры

| Входные данные               | Выход | Пояснение                                      |
| ---------------------------- | ----- | ---------------------------------------------- |
| `nums = [1,1,1], k = 2`      | `2`   | `[1,1]` встречается дважды (индексы 0–1 и 1–2) |
| `nums = [1,2,3], k = 3`      | `2`   | `[1,2]` и `[3]`                                |
| `nums = [1], k = 0`          | `0`   | нет подмассива с суммой 0                      |
| `nums = [-1, -1, 1], k = 0`  | `1`   | `[-1, 1]` на индексах 1–2                      |

### Решение

- Идея: будем проходить по массиву и поддерживать текущую префиксную сумму `prefix`. Если в какой-то момент разность `prefix - k` уже встречалась ранее как префиксная сумма, то все подмассивы, начинающиеся после того вхождения и заканчивающиеся в текущем индексе, имеют сумму `k`. Для быстрого подсчёта храним в хэш-таблице частоты всех встреченных префиксных сумм. Изначально кладём `0: 1`, чтобы учесть подмассивы, начинающиеся с самого начала.
- Сложность: O(n) по времени, O(n) по памяти (для хэш-таблицы).

```python
from collections import defaultdict

def subarrayEqK(nums: List[int], k: int) -> int:
    hashmap = defaultdict(int, {0: 1})
    prefix, res = 0, 0
    for num in nums:
        prefix += num
        if prefix - k in hashmap:
            res += hashmap[prefix - k]
        hashmap[prefix] += 1
    return res
```

## Пропавший номер

#сложная

Руслан — хакер-энтузиаст, который пишет систему самодиагностики для своего "умного" роутера. Роутер каждую миллисекунду присылает массив целых чисел `nums` — результаты мгновенных проверок портов. Руслану нужно мгновенно находить этот номер, чтобы успеть закрыть уязвимость до атаки.

Ваша задача: напишите алгоритм, который по неотсортированному массиву целых чисел `nums` возвращает наименьшее положительное целое число, отсутствующее в массиве. Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти (не считая входных данных).
### Примеры

| Входные данные         | Выходные данные |
| ---------------------- | --------------- |
| `nums = [1,2,0]`       | `3`             |
| `nums = [3,4,-1,1]`    | `2`             |
| `nums = [7,8,9,11,12]` | `1`             |
### Решение

- Решение на $O(n \cdot \log n)$ операций и $O(1)$ памяти: Тривиальная сортировка и линейная прогонка
- Решение за  $O(n)$ операций и $O(1)$ памяти: Заметим, что решение никогда не превосходит N+1. Пройдём по массиву и поставим каждый элемент на индекс, равный значению элемента (элементы со значением, выходящим за пределы индекса, можно отбросить). Тогда при линейной прогонке первый элемент, который не совпадает со своей позицией (или N+1, если такого элемента нет) будет пропущен

```Python
def solution(nums: List[int]) -> int:
    n = len(nums)

    for i in range(n):
        if nums[i] <= 0 or nums[i] > n:
            nums[i] = n + 1

    for i in range(n):
        num = abs(nums[i])
        if 1 <= num <= n:
            nums[num - 1] = -abs(nums[num - 1])

    for i in range(n):
        if nums[i] > 0:
            return i + 1

    return n + 1
```
## Сладкое королевство

#сложная

В Сладком Королевстве наступает Праздник Первого Урожая, и король по традиции угощает всех детей волшебными леденцами. На главной площади выстроились в ряд `n` маленьких волшебников, каждому из которых присвоен **рейтинг мастерства** — целое число в массиве `ratings`. От этого рейтинга зависит, как много сладостей достанется ребёнку. Древний указ гласит:

- Каждый ребёнок должен получить **минимум один леденец**.
- Дети с более высоким рейтингом, чем у соседа, должны получить **строго больше леденцов**, чем этот сосед.

Главный казначей хочет выдать как можно меньше сладостей, но полностью соблюсти королевский указ. Помогите ему подсчитать **минимальное суммарное количество леденцов**, которое необходимо подготовить.

### Примеры

| Входные данные (`ratings`) | Выход (минимальное число леденцов) |
| -------------------------- | ---------------------------------- |
| `[1, 0, 2]`                | `5`                                |
| `[1, 2, 2]`                | `4`                                |
| `[1, 3, 2, 1]`             | `7`                                |
### Решение

- **Решение 1: два прохода с дополнительной памятью O(n)**  
    Сначала выдаём каждому по 1 леденцу.  
    Проход слева направо: если рейтинг выше левого соседа, даём на 1 больше, чем у соседа.  
    Проход справа налево: если рейтинг выше правого соседа, берём максимум из уже имеющегося количества и `candies[i+1] + 1`.  
    Сумма всех значений — ответ.
    

```python
from typing import List
def candy(ratings: List[int]) -> int:
    n = len(ratings)
    candies = [1] * n
    # проход слева направо
    for i in range(1, n):
        if ratings[i] > ratings[i - 1]:
            candies[i] = candies[i - 1] + 1
    # проход справа налево
    for i in range(n - 2, -1, -1):
        if ratings[i] > ratings[i + 1]:
            candies[i] = max(candies[i], candies[i + 1] + 1)
    return sum(candies)
```

- **Решение 2: один проход, память O(1)**  
    Идём по ряду детей, отслеживая текущие «горки» (возрастающие отрезки – `peak`) и «спуски» (убывающие – `valley`).  
    Каждый раз, поднимаясь в горку, добавляем к сумме столько леденцов, на сколько шагов поднялись; спускаясь – аналогично увеличиваем сумму. В конце каждой пары «горка‑спуск» вычитаем меньшее из значений `peak` и `valley`, потому что пик был учтён дважды – и как конец подъёма, и как начало спуска.  
    Равные соседние рейтинги просто прерывают текущую последовательность.
    

```python
from typing import List
def candy(ratings: List[int]) -> int:
    n = len(ratings)
    total = n          # каждый получит минимум 1
    i = 1
    while i < n:
        if ratings[i] == ratings[i - 1]:
            i += 1
            continue
        peak = 0
        while i < n and ratings[i] > ratings[i - 1]:
            peak += 1
            total += peak
            i += 1
        if i == n:
            return total
        valley = 0
        while i < n and ratings[i] < ratings[i - 1]:
            valley += 1
            total += valley
            i += 1
        total -= min(peak, valley)
    return total
```

Оба решения удовлетворяют временной сложности O(n), но второе экономит память, сводя дополнительный расход к константному.
## Древняя мозаика

#сложная  

Археолог Изабелла обнаружила в заброшенном храме огромную напольную мозаику. Время не пощадило шедевр: многие плитки выпали, и теперь мозаика представляет собой матрицу, где `'1'` — целая плитка, а `'0'` — пустота. Изабелла решает вырезать из мозаики **максимальный прямоугольный фрагмент**, полностью состоящий из целых плиток, чтобы перевезти его в музей. Ваша задача — по заданной бинарной матрице определить наибольшую возможную площадь такого прямоугольника.

Дана матрица `matrix` размера `M x N`, каждый элемент которой — символ `'0'` или `'1'`.  
Верните **максимальную площадь прямоугольника**, заполненного только `'1'` и выровненного по строкам и столбцам.

### Примеры

| Входные данные                                                                                                               | Выход |
| ---------------------------------------------------------------------------------------------------------------------------- | ----- |
| `matrix = [`<br>`["1","0","1","0","0"],`<br>`["1","0","1","1","1"],`<br>`["1","1","1","1","1"],`<br>`["1","0","0","1","0"]]` | `6`   |
| `matrix = [["1","1"],`<br>`["1","1"]]`                                                                                       | `4`   |
| `matrix = [["0"]]`                                                                                                           | `0`   |
| `matrix = [["1"]]`                                                                                                           | `1`   |

### Решение

- **Идея:** Сначала преобразуем каждую строку в префиксные ширины: для каждой ячейки храним количество подряд идущих единиц слева, включая саму ячейку. Затем для каждого столбца `j` перебираем все строки `i` как потенциальную «опорную» строку. Начиная с текущей ширины, расширяем прямоугольник вниз и вверх, корректируя ширину как минимум всех охваченных строк. На каждом шаге обновляем ответ площадью `ширина × высота`.

- **Сложность:** O(M²·N) по времени, O(1) дополнительной памяти (входная матрица может использоваться как рабочий массив).

```python
from typing import List

class Solution:
    def maximalRectangle(self, matrix: List[List[str]]) -> int:
        if not matrix or not matrix[0]:
            return 0

        M, N = len(matrix), len(matrix[0])

        # Преобразуем символы в целые и строим префиксные ширины в каждой строке
        for i in range(M):
            for j in range(N):
                matrix[i][j] = int(matrix[i][j])
            for j in range(1, N):
                if matrix[i][j] == 1:
                    matrix[i][j] += matrix[i][j - 1]

        max_area = 0

        # Фиксируем столбец
        for j in range(N):
            for i in range(M):
                width = matrix[i][j]
                if width == 0:
                    continue

                # Расширение вниз
                cur_width = width
                k = i
                while k < M and matrix[k][j] > 0:
                    cur_width = min(cur_width, matrix[k][j])
                    height = k - i + 1
                    max_area = max(max_area, cur_width * height)
                    k += 1

                # Расширение вверх
                cur_width = width
                k = i
                while k >= 0 and matrix[k][j] > 0:
                    cur_width = min(cur_width, matrix[k][j])
                    height = i - k + 1
                    max_area = max(max_area, cur_width * height)
                    k -= 1

        return max_area
```

## Заклинание древних рун

#сложная

Волшебница Элара обнаружила в старой башне Великий Свиток — бесконечную строку рун, в которой сокрыта магия всех известных заклинаний. Чтобы сотворить чары, ей нужен самый короткий непрерывный фрагмент свитка, где встречаются **все** руны из определённого заклинания (порядок не важен, но важно точное количество одинаковых рун). Свиток записан строкой `s`, а заклинание — строкой `t`. Элара просит вас найти этот фрагмент за время одного прочтения свитка, ведь иначе магия рассеется.

Ваша задача: напишите алгоритм, который для заданных строк `s` и `t` возвращает **минимальную по длине подстроку** `s`, содержащую все символы из `t` (с учётом их кратности). Если таких подстрок несколько, можно вернуть любую. Если подходящей подстроки нет, верните пустую строку `""`.  
Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти (алфавит ограничен).

### Примеры

| Входные данные (s, t)            | Выходные данные |
| -------------------------------- | --------------- |
| `s = "ADOBECODEBANC"`, `t = "ABC"` | `"BANC"`        |
| `s = "a"`, `t = "a"`             | `"a"`           |
| `s = "a"`, `t = "aa"`            | `""`            |
| `s = "ab"`, `t = "b"`            | `"b"`           |
| `s = "abc"`, `t = "cba"`         | `"abc"`         |

### Решение

- **Идея:** Метод скользящего окна с двумя указателями `start` и `end`. Поддерживаем счётчик `count` — сколько символов из `t` ещё не покрыто в текущем окне. Расширяем окно, двигая `end`, и уменьшаем счётчик частот, пока все символы `t` не появятся (`count == 0`). Тогда пытаемся сузить окно с помощью `start`, обновляя минимальную длину и стартовый индекс. При удалении символа из окна, если его частота становится критической, увеличиваем `count`. Продолжаем до конца строки.
- **Сложность:** O(n) по времени, O(1) по памяти (массив фиксированного размера 128 для ASCII).

```python
def solution(s: str, t: str) -> str:
    if not s or not t or len(s) < len(t):
        return ""

    # массив частот символов заклинания t
    freq = [0] * 128
    for c in t:
        freq[ord(c)] += 1

    count = len(t)          # сколько символов ещё нужно покрыть
    start = 0
    min_len = float('inf')
    min_start = 0

    for end in range(len(s)):
        if freq[ord(s[end])] > 0:
            count -= 1          # нашли нужный символ
        freq[ord(s[end])] -= 1  # забираем его в окно (может стать отрицательным)

        while count == 0:       # окно содержит все символы t
            if end - start + 1 < min_len:
                min_len = end - start + 1
                min_start = start

            if freq[ord(s[start])] == 0:
                count += 1      # теряем нужный символ
            freq[ord(s[start])] += 1
            start += 1

    return "" if min_len == float('inf') else s[min_start:min_start + min_len]
```
