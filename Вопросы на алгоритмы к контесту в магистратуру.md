## Шифр первокурсника

#лёгкая

На факультете шифрования Центрального университета первокурсникам выдали необычное задание: научиться мгновенно переворачивать каждое слово в университетских объявлениях, чтобы, по легенде, защитить расписание от злых духов-спойлеров. Порядок слов остаётся неизменным, а вот буквы внутри каждого слова встают задом наперёд. На вход подаётся строка, и вам нужно вернуть её «отзеркаленную» версию, сохранив все пробелы и последовательность слов.

Ваша задача: реализовать алгоритм, который переворачивает каждое слово в строке `s` (слова разделены ровно одним пробелом), не меняя порядка слов. Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти, не считая выходной строки.

### Примеры

| Входные данные                                     | Выходные данные                                |
| -------------------------------------------------- | ---------------------------------------------- |
| `s = "Вот бы поступить в Центральный Университет"` | `"тоВ ыб ьтипутсоп в йыньлортнеЦ тетисревинУ"` |
| `s = "a b c d"`                                    | `"a b c d"`                                    |
| `s = "hello"`                                      | `"olleh"`                                      |

### Решение

- **Идея:** разделяем строку по пробелам на слова, каждое слово переворачиваем с помощью среза `[::-1]` (или двух указателей), затем склеиваем обратно через пробел.
- **Сложность:** O(n) по времени (каждый символ обрабатывается дважды) и O(n) памяти для результирующей строки. Если не учитывать выходную строку, дополнительной памяти используется O(1).

```python
def reverseWords(s: str) -> str:
    return ' '.join(word[::-1] for word in s.split())
```

## Слияние караванов

#легкая

Шейх Халид получил два каравана с золотыми слитками, пронумерованными по весу. Каждый караван уже отсортирован по весу от лёгких слитков к тяжёлым. Первый караван (`nums1`) идёт с запасом — в конце у него достаточно свободных верблюдов (заполненных нулями), чтобы разместить всё золото второго каравана. Шейх приказал слить оба каравана прямо на ходу, не используя лишнего снаряжения, чтобы итоговый караван остался строго отсортированным по весу.

Вам даны два целочисленных массива `nums1` и `nums2`, отсортированные по неубыванию, и два числа `m` и `n` — количество значимых элементов в `nums1` и `nums2` соответственно. Массив `nums1` имеет размер `m + n`, где первые `m` элементов — значимые, а остальные `n` заполнены нулями. Объедините элементы `nums2` в `nums1` **на месте**, чтобы итоговый массив остался отсортированным. Алгоритм должен работать за **O(m + n)** времени и **O(1)** дополнительной памяти.

### Примеры

| Входные данные (nums1, m, nums2, n)               | Результат (модифицированный nums1) |
| ------------------------------------------------- | --------------------------------- |
| `nums1 = [1,2,3,0,0,0], m = 3, nums2 = [2,5,6], n = 3` | `[1,2,2,3,5,6]`                  |
| `nums1 = [1], m = 1, nums2 = [], n = 0`           | `[1]`                            |
| `nums1 = [0], m = 0, nums2 = [1], n = 1`          | `[1]`                            |
| `nums1 = [4,5,6,0,0,0], m = 3, nums2 = [1,2,3], n = 3` | `[1,2,3,4,5,6]`                  |

### Решение

- **Простое решение (O((m+n) log(m+n))):** скопировать `nums2` в конец `nums1` и отсортировать массив целиком. Не требует дополнительной памяти под новый массив, но модифицирует тот же.
- **Оптимальное решение (O(m+n), O(1)):** идём с конца двумя указателями — один на последний значимый элемент `nums1` (индекс `m-1`), второй на последний элемент `nums2` (индекс `n-1`). Сравниваем их и ставим больший в позицию `m+n-1`, сдвигая соответствующий указатель. Так заполняем массив от конца к началу, не затирая необработанные данные.

```python
from typing import List

def merge(nums1: List[int], m: int, nums2: List[int], n: int) -> None:
    i, j, k = m - 1, n - 1, m + n - 1
    while i >= 0 and j >= 0:
        if nums1[i] > nums2[j]:
            nums1[k] = nums1[i]
            i -= 1
        else:
            nums1[k] = nums2[j]
            j -= 1
        k -= 1
    # Если остались элементы в nums2, переносим их
    while j >= 0:
        nums1[k] = nums2[j]
        j -= 1
        k -= 1
```
## Зеркальное Древо

#легкая

В Зачарованном Лесу растёт особое Древо-Близнец. Легенда гласит, что если дерево зеркально симметрично, то оно служит порталом в параллельный мир. Эльфийский старейшина хочет проверить, является ли данное бинарное дерево волшебным порталом — для этого нужно убедиться, что оно симметрично относительно своего центра (левое поддерево является зеркальным отражением правого).

Ваша задача: напишите функцию, которая по корню бинарного дерева `root` определяет, является ли дерево **зеркально симметричным**.

### Примеры

**Пример 1**

```
          1
         / \
        2   2
       / \ / \
      3  4 4  3
```
Выход: `True` — левая половина отражает правую.

**Пример 2**

```
      1
     / \
    2   2
     \   \
      3   3
```
Выход: `False` — правый потомок левого поддерева не равен левому потомку правого поддерева.

**Пример 3**

```
          1
         / \
        2   2
       /     \
      3       3
```
Выход: `True`

### Решение

- **Идея:** рекурсивно сравниваем два поддерева — левое и правое. Два узла зеркальны, если их значения равны, левый потомок левого узла зеркален правому потомку правого, а правый потомок левого — левому потомку правого. Базовые случаи: оба `None` → `True`, один `None` → `False`.
- **Сложность:** O(n) по времени (посещаем каждый узел один раз), O(h) по памяти из-за стека рекурсии, где h — высота дерева.

```python
from typing import Optional

# Определение узла дерева
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def solution(root: Optional[TreeNode]) -> bool:
    def isMirror(left, right):
        if not left and not right:
            return True
        if not left or not right:
            return False
        return (
            left.val == right.val 
            and isMirror(left.left, right.right) 
            and isMirror(left.right, right.left)
        )
    
    if not root:
        return False  # по условию: пустое дерево не считается симметричным
    return isMirror(root.left, root.right)
```



## Страницы пиратского дневника

#легкая

Капитан Блэк нашёл в затопленной каюте потрёпанный дневник старого штурмана. Все уцелевшие страницы пронумерованы, аккуратно отсортированы, и номера уникальны — пират это проверил лично. Теперь ему нужно сжать записи в судовом журнале: выписать все непрерывные диапазоны страниц, чтобы сэкономить чернила и место. Отдельную страницу записывают как сам номер, а диапазон — как «начало → конец». Например, страницы 0, 1, 2, 4, 5 и 7 превратятся в запись `"0->2"`, `"4->5"`, `"7"`.

Ваша задача: получив отсортированный массив уникальных целых чисел `nums`, вернуть список строк, представляющих **минимальный набор непрерывных диапазонов**, покрывающих все числа. Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти (не считая выходного списка).

### Примеры

|Входные данные|Выходные данные|
|---|---|
|`nums = [0,1,2,4,5,7]`|`["0->2","4->5","7"]`|
|`nums = [1,3]`|`["1","3"]`|
|`nums = [1,2,3,5,6,8]`|`["1->3","5->6","8"]`|
|`nums = [10]`|`["10"]`|

### Решение

- **Идея:** линейным проходом находим начало `start` диапазона и, пока следующий элемент отличается от текущего ровно на 1, расширяем конец `j`. После этого записываем диапазон в соответствующем формате и перепрыгиваем на следующий за концом элемент.
- **Сложность:** O(n) по времени, O(1) дополнительной памяти (не считая результирующего списка).

```python
def solution(nums: List[int]) -> List[str]:
    i, result = 0, []
    while i < len(nums):
        start, j = nums[i], i
        while j + 1 < len(nums) and nums[j+1] == nums[j] + 1:
            j += 1
        result.append(f"{start}" if nums[j] == start else f"{start}->{nums[j]}")
        i = j + 1
    return result
```


## Золотые слитки

#легкая

Гном-рудокоп Бодрик нашёл в заброшенной штольне старый сундук с золотыми слитками. Каждый слиток имеет свой вес, записанный на нём древними цифрами. Сундук заперт хитрым замком: чтобы открыть, нужно выбрать **ровно два слитка**, суммарный вес которых в точности равен магическому числу `target`, выбитому на крышке. Бодрик торопится до заката, поэтому просит вас найти **индексы** этих двух слитков в массиве `nums` как можно быстрее. Гарантируется, что ровно одно решение существует, и нельзя брать один и тот же слиток дважды.

Ваша задача: напишите алгоритм, который принимает массив целых чисел `nums` и целое число `target`, и возвращает индексы двух элементов, сумма которых равна `target`. Вернуть индексы можно в любом порядке. Постарайтесь реализовать решение, работающее быстрее, чем полный перебор.

### Примеры

| Входные данные (nums, target)       | Выходные данные |
| ----------------------------------- | --------------- |
| `nums = [2,7,11,15]`, `target = 9`  | `[0,1]`         |
| `nums = [3,2,4]`, `target = 6`      | `[1,2]`         |
| `nums = [3,3]`, `target = 6`        | `[0,1]`         |
| `nums = [1,5,7]`, `target = 12`     | `[1,2]`         |

### Решение

- **Полный перебор за O(n²):** Проверяем каждую пару элементов вложенными циклами. Самое простое, но медленное решение
  
```Python
def twoSum_brute(nums: List[int], target: int) -> List[int]:
    n = len(nums)
    for i in range(n - 1):
        for j in range(i + 1, n):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```

-   **Двухпроходная хеш-таблица за O(n):** На первом проходе запоминаем позицию каждого элемента в словаре. На втором — для каждого элемента проверяем, есть ли его дополнение `target - nums[i]` в словаре, и чтобы это не был тот же самый индекс.
  
  ```Python
def twoSum_two_pass(nums: List[int], target: int) -> List[int]:
    num_map = {}
    for i, num in enumerate(nums):
        num_map[num] = i

    for i, num in enumerate(nums):
        complement = target - num
        if complement in num_map and num_map[complement] != i:
            return [i, num_map[complement]]
    return []
  ```
  
- **Однопроходная хеш-таблица за O(n):** Совмещаем построение словаря и поиск. Проходя по массиву, для каждого элемента проверяем, встречался ли ранее его комплемент. Если да — сразу возвращаем ответ, если нет — добавляем текущий элемент в словарь.

```python
def twoSum(nums: List[int], target: int) -> List[int]:
    num_map = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in num_map:
            return [num_map[complement], i]
        num_map[num] = i
    return []
```


## Тайное послание магистра

#легкая

Магистр ордена «Двух Зеркал» перехватил зашифрованное послание — длинную строку `s`, полную отвлекающих символов: знаков препинания, пробелов и прочей мишуры. Истинное сообщение читается одинаково слева направо и справа налево, если отбросить всё лишнее и не различать заглавные и строчные буквы. Только такое зеркальное послание откроет врата ордена. Помогите магистру проверить подлинность строки.

Ваша задача: напишите алгоритм, который принимает строку `s` и возвращает `true`, если она является **палиндромом** после приведения всех букв к нижнему регистру и игнорирования всех символов, кроме букв и цифр. В противном случае верните `false`.

### Примеры

| Входные данные (s)                          | Выходные данные |
| ------------------------------------------- | --------------- |
| `s = "A man, a plan, a canal: Panama"`      | `true`          |
| `s = "race a car"`                          | `false`         |
| `s = " "`                                   | `true`          |
| `s = "0P"`                                  | `false`         |
| `s = "Madam, in Eden I'm Adam"`             | `true`          |

### Решение

- **Идея:** Метод двух указателей `l` и `r`, идущих навстречу друг другу от краёв строки. На каждом шаге пропускаем все не-букво-цифры. Затем сравниваем символы, приведённые к нижнему регистру. Если хоть раз не совпали — возвращаем `false`. Если указатели пересеклись — строка является палиндромом.
- **Сложность:** O(n) по времени, O(1) дополнительной памяти.

```python
def validPalindrome(s: str) -> bool:
    l, r = 0, len(s) - 1
    while l < r:
        # пропускаем не буквенно-цифровые символы
        while l < r and not s[l].isalnum():
            l += 1
        while l < r and not s[r].isalnum():
            r -= 1
        if s[l].lower() != s[r].lower():
            return False
        l += 1
        r -= 1
    return True
```

## Зеркальный коридор

#легкая

Авантюристка Лин попала в древний храм, полный магических дверей. Каждая дверь указывает на следующую, образуя односвязный проход. Согласно легенде, чтобы выбраться наружу, нужно пройти все двери в строго обратном порядке — иначе храм обрушится. Лин стоит у первой двери — головы списка `head`, — и ей срочно нужна карта обратного маршрута, где каждая дверь будет указывать на предыдущую.

Ваша задача: напишите алгоритм, который принимает голову `head` односвязного списка и возвращает голову **развёрнутого** списка (то есть последний элемент становится первым, а все ссылки «следующий» переворачиваются). Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти.

### Примеры

| Входные данные (head)         | Выходные данные               |
| ----------------------------- | ----------------------------- |
| `[1, 2, 3, 4, 5]`             | `[5, 4, 3, 2, 1]`            |
| `[1, 2]`                      | `[2, 1]`                      |
| `[]`                          | `[]`                          |
| `[42]`                        | `[42]`                        |

*Примечание:* в примерах списки представлены в виде массива значений для наглядности.

### Решение

- **Идея:** Итеративный метод с тремя указателями. Заводим `prev = None` (будущая голова развёрнутого списка) и `current = head`. На каждом шаге сохраняем `nxt = current.next`, затем разворачиваем связь `current.next = prev`. После этого сдвигаем `prev` на `current`, а `current` на `nxt`. Когда `current` становится `None`, `prev` указывает на новую голову.
- **Сложность:** O(n) по времени, O(1) дополнительной памяти.

```python
def reverseLinkedList(head):
    prev, current = None, head
    while current:
        nxt = current.next      # запоминаем следующую дверь
        current.next = prev     # разворачиваем указатель
        prev = current          # идём вперёд
        current = nxt
    return prev
```

## Сила рунного камня

#легкая

В подземельях древних гномов археолог Лина нашла магический рунный камень — длинную цепочку символов, высеченных на граните. Сила камня зависит от самой длинной непрерывной последовательности одинаковых рун: чем длиннее цепочка, тем мощнее заклинание можно сотворить. Лине нужно вычислить эту максимальную длину, чтобы понять, стоит ли брать камень с собой.

Ваша задача: для заданной строки `s` найти длину **самой длинной подстроки, состоящей из одного и того же символа** (идущих подряд). Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти.

### Примеры

| Входные данные               | Выходные данные | Пояснение               |
| ---------------------------- | --------------- | ----------------------- |
| `s = "leetcode"`             | `2`             | `"ee"` — две 'e' подряд |
| `s = "abbcccddddeeeeedcba"`  | `5`             | `"eeeee"` — пять 'e'    |
| `s = "a"`                    | `1`             | один символ             |
| `s = "abcde"`                | `1`             | все символы разные      |
| `s = "aaabbb"`               | `3`             | `"aaa"` — три 'a'       |

### Решение

- **Идея:** Линейный проход с двумя указателями или итеративный подсчёт текущей длины последовательности одинаковых символов. Проходим строку, увеличиваем счётчик `curr`, пока следующий символ совпадает с текущим. Как только последовательность обрывается, обновляем `max_len` и сбрасываем `curr` в 1. Продолжаем до конца строки.
- **Сложность:** O(n) по времени, O(1) по памяти.

```python
def solution(s: str) -> int:
    curr = 1
    max_len = 0
    i = 0
    while i < len(s):
        while i + 1 < len(s) and s[i + 1] == s[i]:
            curr += 1
            i += 1
        max_len = max(max_len, curr)
        i += 1
        curr = 1
    return max_len
```


## Голосование за блюдо дня

#легкая

Антон работает офис-менеджером в крупной IT-компании. Каждую пятницу сотрудники голосуют за одно блюдо, которое закажут на обед. Голосование проходит в цифровом виде, и система присылает Антону длинный массив номеров блюд — `nums`, где каждый элемент — это код блюда из фиксированного меню. По регламенту, чтобы заказ был принят, какое-то одно блюдо должно набрать **строго больше половины** всех голосов. Гарантируется, что такое блюдо всегда есть. Антону нужно как можно быстрее определить победителя, чтобы успеть оформить заказ до закрытия кухни.

Ваша задача: написать алгоритм, который в массиве целых чисел `nums` длиной `n` находит **элемент, встречающийся более `n/2` раз** (majority element). Алгоритм должен быть эффективным по времени и, по возможности, не расходовать лишнюю память.

### Примеры

| Входные данные (nums)        | Выходные данные | Пояснение                                 |
| ---------------------------- | --------------- | ----------------------------------------- |
| `nums = [3,2,3]`             | `3`             | 3 встречается 2 раза (2 > 3/2)            |
| `nums = [2,2,1,1,1,2,2]`     | `2`             | 2 встречается 4 раза (4 > 7/2)            |
| `nums = [5]`                 | `5`             | один элемент — автоматически большинство  |
| `nums = [1,1,2,1,2,2,2]`    | `2`             | 2 встречается 4 раза из 7                 |

### Решение

- **Сортировка за O(n log n):** Если массив отсортировать, то элемент, занимающий позицию `n/2`, гарантированно будет мажоритарным — ведь он встречается более половины раз и «перекроет» середину. Просто, но требует модификации исходного порядка.
- **Хеш-таблица за O(n):** Подсчитываем частоты всех элементов в словаре и ищем тот, у которого значение больше `n/2`. Надёжно, но требует O(n) дополнительной памяти.
- **Алгоритм Бойера-Мура за O(n) и O(1):** Используем «голосование»: поддерживаем `candidate` и `count`. Когда `count == 0`, назначаем текущий элемент кандидатом. Если очередной элемент равен кандидату — увеличиваем счётчик, иначе уменьшаем. В конце кандидат и есть искомый элемент. Самый экономный по памяти метод.

```python
from typing import List
from collections import defaultdict

# Решение 1: Сортировка O(n log n)
def majorityElement_sort(nums: List[int]) -> int:
    nums.sort()
    return nums[len(nums) // 2]

# Решение 2: Хеш-таблица O(n)
def majorityElement_hash(nums: List[int]) -> int:
    counts = defaultdict(int)
    for num in nums:
        counts[num] += 1
    threshold = len(nums) // 2
    for num, cnt in counts.items():
        if cnt > threshold:
            return num
    return -1  # по условию гарантируется, что не понадобится

# Решение 3: Алгоритм Бойера-Мура O(n), O(1)
def majorityElement_moore(nums: List[int]) -> int:
    count = 0
    candidate = 0
    for num in nums:
        if count == 0:
            candidate = num
        if num == candidate:
            count += 1
        else:
            count -= 1
    return candidate
```

## Логи транзакций

#легкая

Дмитрий — разработчик в финтех-стартапе — отлаживает новую версию платёжного шлюза. Все транзакции за день записываются в односвязный список: каждый узел хранит сумму операции и ссылку на следующую. Технический аудит требует, чтобы последовательность транзакций была строго симметричной — это значит, что список должен быть **палиндромом**. Дмитрию нужно проверить это как можно быстрее и не запрашивая дополнительную память под копию данных.

Ваша задача: напишите алгоритм, который принимает голову односвязного списка `head` и возвращает `true`, если значения узлов образуют палиндром, и `false` в противном случае. Алгоритм должен работать за **O(n)** времени и использовать **O(1)** дополнительной памяти (не считая входных данных).

### Примеры

| Входные данные (список)    | Выходные данные |
| -------------------------- | --------------- |
| `1 → 2 → 2 → 1`           | `true`          |
| `1 → 2`                   | `false`         |
| `1 → 2 → 1`               | `true`          |
| `пустой список`           | `true`          |
| `5`                       | `true`          |

### Решение

- **Идея:** Найти середину списка с помощью двух указателей (`slow` и `fast`). Развернуть вторую половину списка на месте, не выделяя новую память. Затем идти от начала и от перевёрнутой середины и сравнивать значения. Если все совпали — это палиндром.
- **Сложность:** O(n) по времени, O(1) по памяти — мы только переставляем указатели внутри существующих узлов.

```python
from typing import Optional

# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def solution(head: Optional[ListNode]) -> bool:
    # 1. Поиск середины
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    # 2. Разворот второй половины
    prev = None
    while slow:
        nxt = slow.next
        slow.next = prev
        prev = slow
        slow = nxt

    # 3. Сравнение первой половины с перевёрнутой второй
    left, right = head, prev
    while right:  # правая половина может быть короче на 1 элемент
        if left.val != right.val:
            return False
        left = left.next
        right = right.next
    return True
```
